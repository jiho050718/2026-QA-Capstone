## 1. 프롬프트 엔지니어링 개요

### 🔹 정의
프롬프트 엔지니어링은 AI 모델로부터 최적의 응답을 얻기 위해 입력값(프롬프트)을 전략적으로 설계, 최적화하는 기술입니다. 단순히 '질문'을 던지는 행위를 넘어, AI의 논리적 추론을 유도하는 전문적인 설계 과정입니다.

### 🔹 중요성
* **정확성:** 사용자의 의도를 명확히 전달하여 할루시네이션(환각)을 방지합니다.
* **효율성:** 반복적인 질문 없이 한 번에 고품질의 결과물을 도출합니다.
* **협업:** AI를 단순한 도구가 아닌 비즈니스 및 학습의 파트너로 활용하게 합니다.

## 2. 대화 질문 설계의 10가지 핵심 요소

AI에게 명확한 가이드를 제공하기 위해 프롬프트에 포함해야 할 상세 구성 요소입니다.

| 구분 | 요소 (Element) | 상세 설명 |
| :--- | :--- | :--- |
| **1** | **목적 (Objective)** | 최종적으로 얻고자 하는 결과물의 명확한 목표 |
| **2** | **상황 (Context)** | 질문 배경, 관련 정보, 필요한 제약 조건 설명 |
| **3** | **역할 (Role)** | AI에게 부여하는 특정 전문 페르소나 (예: 교수, 코드 리뷰어) |
| **4** | **청중 (Audience)** | 답변을 수신할 대상의 수준 (예: 비전공자, 전문가) |
| **5** | **예시 (Examples)** | 원하는 출력 형태나 내용의 샘플 데이터 제공 |
| **6** | **형식 (Format)** | 표(Table), 리스트, JSON, 마크다운 등 출력 구조 지정 |
| **7** | **키워드 (Keywords)** | 답변에 반드시 포함되어야 할 핵심 용어 지정 |
| **8** | **제한 (Limitations)** | 답변 길이, 금지 단어, 시간적 범위 등 제약 설정 |
| **9** | **어조 (Tone)** | 답변의 분위기 (예: 격식 있는, 친절한, 비판적인) |
| **10** | **부정 (Negation)** | 답변에서 반드시 제외해야 할 내용 명시 |

## 3. 효과적인 질문 전략 (단계별 & 맥락 중심)

### 🧠 단계별 생각하기 (Chain of Thought)
복잡한 문제를 해결할 때 AI에게 **"단계적으로 생각해보자(Let's think step by step)"**라고 지시합니다. 이는 AI가 내부적으로 중간 추론 과정을 거치게 하여 수학적 계산이나 논리적 오류를 크게 줄여줍니다.

### 📍 구체적 맥락 제공
추상적인 질문 대신 구체적인 상황을 설정합니다.
* **Bad:** "세탁기 고르는 법 알려줘."
* **Good:** "1인 가구이고 오피스텔에 거주 중이야. 층간 소음이 적고 건조 기능이 포함된 10kg 이하 세탁기 추천 리스트를 장단점 위주로 정리해줘."

## 4. 프롬프트 유형: Zero-shot, One-shot, Few-shot

AI에게 제공하는 예시의 개수에 따라 학습 및 응답 성능이 달라집니다.

1. **Zero-shot (제로샷):** 예시 없이 바로 지시를 내리는 방식. AI의 기본 학습 지식에 의존합니다.
2. **One-shot (원샷):** 한 가지의 모범 사례(Example)를 제공하여 출력 형식을 가이드합니다.
3. **Few-shot (퓨샷):** 여러 개의 예시를 제공하여 복잡한 패턴이나 규칙을 학습시킵니다.
   - *사례:* "Q: 머신러닝이란? A: 데이터를 학습하여 패턴을 인식하는 기술입니다. Q: 딥러닝이란? A: [이후 자동 생성]"

## 5. 구조화된 프롬프트를 위한 기호 활용

프롬프트 내에서 섹션을 명확히 구분하기 위해 마크다운 및 구분 기호를 사용합니다.

* **구분자 (Delimiters):** `"""` (큰따옴표 세 개)나 `---`를 사용하여 지시문과 본문 텍스트를 분리합니다.
* **계층 구조:** `#`를 사용하여 제목과 본문을 구분함으로써 AI가 정보의 중요도를 인식하게 합니다.
* **코드 블록:** ` ``` `를 활용하여 프로그래밍 코드나 데이터 구조를 명시합니다.
